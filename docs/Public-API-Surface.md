# Public API surface (Phase 1.3)

This is a **map** of supported entry points, not a substitute for Xcode DocC. Prefer **`import Fountain`** in apps (umbrella); use **`import FountainCore`** or **`import FountainHTML`** only when you want a slimmer dependency graph.

**Phase 1.3 status:** **Complete** for documentation and stability expectations below. **Phase 7** (SwiftPM test matrix and external-suite tracking) is documented in [Fountain-1.1-Implementation-Roadmap.md](Fountain-1.1-Implementation-Roadmap.md) and [External-Fountain-Test-References.md](External-Fountain-Test-References.md). **Phase 8** (``FountainScriptRendering`` + plaintext / Markdown / JSON / HTML + **FDX / PDF** exporters) is **initial-complete** per the roadmap. **Phase 9** (async full parse + element streaming; **9.4** ``FountainLineToElementIndexMap`` + UTF-16 / ``String/Index`` spans on canonical ``syntheticBodyLineText``; **9.5** ``FountainEditRangeExpansion`` + ``FNScript/parseIncremental(newText:editedUTF16Range:parser:)`` — **full** re-parse + prefix/suffix **ID** merge today, **chunk-only** re-parse still future) is summarized in the roadmap and [Fountain-Incremental-Parse-Spike.md](Fountain-Incremental-Parse-Spike.md). **Phase 10** (SPM default distribution, CI parser/UI boundary + **CoreGraphics/CoreText** allowlist for **FountainCore**, optional Wasm **FountainCore** build) is **initial-complete** per the roadmap — [SPM-Release-Checklist.md](SPM-Release-Checklist.md), [SwiftWasm-Experimental.md](SwiftWasm-Experimental.md), `.github/workflows/swift.yml`. **Phase 11** (Swift **`Regex`** in **`String+Regex.swift`** with a small **`NSRegularExpression`** fallback when Swift cannot compile a pattern, **`FountainRegexes`** body patterns Swift-compatible, **macOS 13 / iOS 16** floor) is **complete (initial)** — [§ Phase 11](Fountain-1.1-Implementation-Roadmap.md#phase-11-regex-modernization-swift-native). **Phase 12** (default **``FountainParsePipeline``** / **``.tokenPipeline``** on ``FNScript`` and async/stream defaults) is **initial-complete** — [§ Phase 12](Fountain-1.1-Implementation-Roadmap.md#phase-12-canonical-state-aware-parser-default-fnscript). **Phase 13** (optional **`FountainUI`** + ``FountainView`` — **SwiftUI** only under **`FountainUI/`**, not in **FountainCore**) is **in progress (initial)** — [§ Phase 13](Fountain-1.1-Implementation-Roadmap.md#phase-13-swiftui-and-fountainui). **`@_spi(…)`** is not applied in source yet; when a subsystem stabilizes, you may narrow visibility without changing runtime behavior.

## API tiers (semver guidance)

| Tier | Treat as | Examples |
|------|-----------|----------|
| **Stable intent** | Avoid breaking changes in patch releases; deprecate before removal. | `FNScript` default initializers (**``.tokenPipeline``** / ``FountainParsePipeline``), `FNElement` (value type: `id` + `elementType` / `elementText` fields), `FountainWriter`, explicit **``.fast``** (``FastFountainParser``) when you need the line-first engine |
| **Preferred interchange** | Additive fields/keys OK in minor releases. | `FountainDocument`, `ScriptElement`, `ScriptElementKind`, `FountainMetadataKey` |
| **Evolving** | May change more freely; document in release notes. | `FountainScriptMetrics` fields, new `FountainScriptRendering` conformers, coarse tokenizer output (`FountainBodyLineTokenizer` / `FountainTokenizedLine` kinds), `FountainScriptElementsBuilder`, incremental **chunk** merge (Phase **9.5** stretch), **`FountainUI`** / ``FountainView`` / ``FountainScriptElementTypography`` (Phase **13** layout and styling) |
| **Experimental** | Not semver-stable; may change or be removed. | Streaming APIs, `FountainMarkdownWriter`, `FountainFDXWriter` / `FountainPDFWriter` export layouts (minimal .fdx structure; PDF base64 via ``FountainScriptRendering`` — treat output format as evolving) |

## FDX and PDF export (consumer contract)

Use this section as the **single** contract for Final Draft and PDF output (see also [SwiftWasm-Experimental.md](SwiftWasm-Experimental.md) for Wasm).

| Writer | Output | How to consume |
|--------|--------|----------------|
| **`FountainFDXWriter`** | **Final Draft `.fdx` XML** (`String`): `FinalDraft` + `Content` + `Paragraph`/`Text`, plus **`<PageLayout>`**, **`<ElementSettings>`** (General, Scene Heading, Action, Character, Dialogue, Parenthetical, Transition), and **`<MoresAndContinueds>`** (industry-style **(MORE)** / **(cont'd)** / scene-continued strings) so Final Draft opens with **standard margins** — still **not** a complete native FD template (no watermark / revision grids). | Save UTF-8 bytes to a `.fdx` file. **Regression:** `Tests/FountainPackageTests/Fixtures/export-golden-minimal.{fountain,fdx}` + ``ExportGoldenFixtureTests/testFDXWriterMatchesGoldenMinimalFixture`` — update the `.fdx` fixture only when intentionally changing export shape. |
| **`FountainPDFWriter`** (**FountainCore**) | **US Letter**, Courier, CoreGraphics + CoreText (`Data` via ``renderPDFData(_:)`` **or** ``renderPDFData(pages:script:)`` for pre-fanned **screenplay pages**). **Headers:** **top-right** page numbers + **top-left** draft title from title-page **Title** key when present. Flat ``renderPDFData(_:)`` wraps overflow internally; **paginated** export uses **Fountain** (see below). | **Always** use ``renderPDFData(_:)`` or ``renderPDFData(pages:script:)`` for files. ``render(_:)`` returns **base64**. **Regression:** ``ExportGoldenFixtureTests``, ``FountainScriptRenderingTests`` (PDFKit). |
| **Paginated PDF** (**Fountain** umbrella only) | ``FountainPDFWriter/renderPDFDataPaginated(script:)`` runs ``FNPaginator`` then ``renderPDFData(pages:script:)`` so **(MORE)** / **(CONT’D)** and slug/page-break rules match in-app pagination. | Requires ``import Fountain`` (not **FountainCore** alone). |
| **Wasm or hosts without CoreGraphics / CoreText** | **`FountainPDFWriter`** is a **stub** (throws ``FountainStubRendererError``); **FDX** still works (Foundation-only). | Use FDX / plaintext / JSON on WebAssembly builds of **FountainCore** (or any compile where CG+CT are unavailable). |

## Stability expectations

- **`fountainDocument`** / **`asFountainDocument()`** build a **new** snapshot each time; each ``ScriptElement`` reuses the corresponding parsed ``FNElement/id`` so IDs stay aligned across repeated snapshots of the same ``FNScript``. For stable JSON bytes, call ``fountainDocumentJSONData(prettyPrinted:)`` once, or hold a single ``FountainDocument`` value while comparing exports.
- **`FNScript`** / **`FNElement`** / **`FountainWriter`** string labels (`elementType`) are **legacy-stable** for existing documents.
- **`FountainDocument`** / **`ScriptElement`** / **`ScriptElementKind`** are the **preferred** interchange shape for JSON and tooling; additive metadata keys may appear in minor releases.
- **`FountainScriptRendering`** conformers may grow; **FDX/PDF** output details (layout, full Final Draft metadata) may evolve in minor releases — decode PDF via ``FountainPDFWriter/renderPDFData(_:)`` when you need `Data`.

## Experimental / evolving

- **`FNScript.scriptElementStream(from:)`** / **`scriptElementStream(fromFile:)`** — full parse then stream (``scriptElementStream(from:)`` uses ``parseStringAsync``; overloads with **`parser:`** mirror sync ``FNScript``); **not** line-incremental (see [Fountain-Incremental-Parse-Spike.md](Fountain-Incremental-Parse-Spike.md)).
- **`FountainEditRangeExpansion`** / **`FNScript.parseIncremental(newText:editedUTF16Range:parser:)`** — **heuristic** UTF-16 invalidation windows + **full** document re-parse with prefix/suffix **``FNElement/id``** preservation; **not** a chunk-only tokenizer warm path yet (same spike doc).
- **`FountainMarkdownWriter`** — lossy projection for tools, not a Fountain spec inverse.

For semantic versioning guidance when you change any of the above, see [SPM-Release-Checklist.md](SPM-Release-Checklist.md).
